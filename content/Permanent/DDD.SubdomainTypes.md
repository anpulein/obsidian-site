## 🎯 Основные понятия

> [!abstract] 💡 Ключевая идея **Subdomain Types** — классификация поддоменов по их стратегической важности и сложности для оптимального распределения ресурсов команды разработки

**Цель классификации:**

- Определить приоритеты инвестиций в разработку
- Выбрать подходящую команду для каждого поддомена
- Принять решения о технологическом стеке
- Оптимизировать архитектурные решения

---

## 🏗️ Типы поддоменов

### 🎖️ Core Subdomain (Основной)

> [!success] 🎯 Core Domain **Сердце бизнеса** — то, что создает конкурентное преимущество и максимальную ценность

|Характеристика|Описание|
|---|---|
|**Сложность**|Высокая бизнес-логика|
|**Важность**|Критическая для бизнеса|
|**Изменения**|Частые, требуют экспертизы|
|**Команда**|Лучшие разработчики|
|**Подход**|Максимальные инвестиции|

**Примеры:**

- Алгоритм рекомендаций в Netflix
- Торговый движок в биржевой системе
- Система ценообразования в авиакомпании

### 🤝 Supporting Subdomain (Поддерживающий)

> [!info] 🔧 Supporting Domain **Поддержка бизнеса** — необходим для работы Core Domain, но не создает конкурентного преимущества

|Характеристика|Описание|
|---|---|
|**Сложность**|Средняя|
|**Важность**|Поддерживающая|
|**Изменения**|Умеренные|
|**Команда**|Опытные разработчики|
|**Подход**|Собственная разработка|

**Примеры:**

- Система уведомлений
- Интеграция с внешними сервисами
- Система отчетности для внутреннего использования

### 🛠️ Generic Subdomain (Общий)

> [!note] ⚙️ Generic Domain **Стандартная функциональность** — решает типовые задачи, доступны готовые решения

|Характеристика|Описание|
|---|---|
|**Сложность**|Низкая/стандартная|
|**Важность**|Вспомогательная|
|**Изменения**|Редкие|
|**Команда**|Junior/средние разработчики|
|**Подход**|Покупка готовых решений|

**Примеры:**

- Аутентификация/авторизация (Auth0, Keycloak)
- Email-сервис (SendGrid, AWS SES)
- Мониторинг (Prometheus, DataDog)

## ⚖️ Критерии классификации

> [!tip] 📊 Матрица оценки Используйте двумерную оценку: **Стратегическая важность** × **Сложность модели**

```
Высокая сложность
     │
     │  Supporting    │    CORE
     │                │
     ├────────────────┼────────────────► Высокая важность
     │                │
     │   Generic      │  Supporting
     │
Низкая сложность
```

### 🎯 Вопросы для классификации

**Core Domain:**

- Это источник конкурентного преимущества?
- Будет ли бизнес страдать без этой функции?
- Требует ли экспертизы предметной области?

**Supporting Domain:**

- Поддерживает ли Core Domain?
- Специфично ли для нашего бизнеса?
- Можно ли купить готовое решение?

**Generic Domain:**

- Это стандартная функция?
- Есть ли готовые продукты на рынке?
- Минимальная ли кастомизация требуется?

## 🎨 Стратегии разработки

|Тип поддомена|Команда|Технологии|Подход к качеству|
|---|---|---|---|
|**Core**|Лучшие эксперты|Передовые, гибкие|Максимальное тестирование|
|**Supporting**|Опытные разработчики|Проверенные, стабильные|Хорошее покрытие тестами|
|**Generic**|Junior/Outsource|Стандартные/готовые|Базовое тестирование|

> [!warning] ⚠️ Частые ошибки
> 
> - Недооценка важности Supporting domains
> - Переоценка сложности Generic domains
> - Распыление лучших ресурсов на неключевые области
> - Попытка сделать всё самостоятельно
