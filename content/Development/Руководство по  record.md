## Краткое описание

В этом уроке разберём, что такое `record` в C# и зачем он нужен. Рассмотрим его применение в Web API проектах для определения DTO, а также в других типах проектов для моделирования данных и упрощения кода.

## Цели обучения

- Понять концепцию `record` в C# и отличия от обычных классов и структур.
- Узнать, как `record` обеспечивает иммутабельность и value-based сравнение.
- Научиться использовать `record` для определения DTO в Web API проектах.
- Изучить сценарии применения `record` в консолях, WPF, приложениях доменной модели.
- Освоить синтаксис записи `record` и лучшие практики проектирования.

- Основы работы с Web API и моделями (DTO, ViewModel).

## Основные понятия

- **record** — новый тип в C# 9+, предназначенный для хранения неизменяемых данных с автоматическим определением сравнения по значению.
- **Value-based Equality** — механизм, при котором объекты считаются равными, если равны все их свойства, а не только ссылочные идентификаторы.
- **Immutable Object** — объект, состояние которого нельзя изменить после создания.
- **Positional record** — запись, у которой свойства заданы в конструкторе через параметры.
- **With-Expression** — синтаксис для копирования record с изменением части его данных.

## Подробный материал

### 1. Что это?

`record` — это новый специализиро­ванный тип в C#, введённый в версии 9. Он похож на класс, но автоматически генерирует методы `Equals` и `GetHashCode`, а также предоставляет удобный синтаксис для создания неизменяемых объектов. Основная идея: сосредоточиться на данных, а не на поведении.

**Пример:**

> Представьте, что вам нужно описать модель пользователя с полями `Id`, `FirstName`, `LastName`. С `record` это можно сделать одной строкой, получив при этом сравнение по значению и неизменяемость.

### 2. Как это работает?

#### 2.1. Синтаксис определения `record`

Есть два основных способа объявить `record`: классический и позиционный.

**Классический record:**

```csharp
public record User
{
    public int Id { get; init; }
    public string FirstName { get; init; }
    public string LastName { get; init; }
}
```

Здесь `init` означает, что свойства можно установить только при инициализации.

**Positional record:**

```csharp
public record User(int Id, string FirstName, string LastName);
```

Этот подход автоматически создаёт свойства с `get; init;`, а также конструктор, равенство и `Deconstruct`.

#### 2.2. Сравнение по значению

По умолчанию записи сравниваются «по содержимому»: если все свойства равны, объекты считаются равными.

```csharp
var u1 = new User(1, "Ivan", "Petrov");
var u2 = new User(1, "Ivan", "Petrov");

Console.WriteLine(u1 == u2); // True, так как свойства совпадают
```

Для классов без `record` выражение `u1 == u2` вернёт `False`, так как сравниваются ссылки.

#### 2.3. Immutability (Неизменяемость)

После создания экземпляра record его свойства нельзя изменить напрямую (из-за `init`). Если нужно получить новый объект с одним изменённым свойством, используют `with`:

```csharp
var u3 = u1 with { LastName = "Sidorov" };
Console.WriteLine(u3.LastName); // Sidorov
Console.WriteLine(u1.LastName); // Petrov (исходный не изменился)
```

#### 2.4. Дополнительные возможности

- **Deconstruct:**
    
    ```csharp
    var (id, first, last) = u1;
    Console.WriteLine($"Id={id}, Name={first} {last}");
    ```
    
- **Наследование record:** Можно унаследовать один `record` от другого.
    
    ```csharp
    public record Person(string FirstName, string LastName);
    public record Employee(int Id, string FirstName, string LastName) : Person(FirstName, LastName);
    ```
    

### 3. Зачем это нужно?

- **Упрощение кода:** Компактная запись моделей с автогенерацией методов сравнения, хеширования и копирования.
- **Правильное сравнение:** Объекты сравниваются по значениям, что важно при работе с DTO и кешированием.
- **Иммутабельность:** Позволяет избежать ошибок, связанных с изменением состояния объектов (потокобезопасность, предсказуемость).
- **Чистая архитектура:** В доменной модели `record` может представлять Value Object, где важны только значения.    

**Пример из Web API:**

> В API часто используют DTO (объекты передачи данных). Если определить DTO как `record`, вы получите удобный синтаксис и гарантированное сравнение данных, что упрощает тестирование и сериализацию.

## Практические примеры / Задачи

1. **Задача 1. Создать DTO для Web API**  
    _Описание:_ Определить модель `ProductDto` с полями `Id`, `Name`, `Price`. Использовать `record` и протестировать сериализацию в JSON.  
    _Ожидаемое решение:_
    
    ```csharp
    public record ProductDto(int Id, string Name, decimal Price);
    ```
    
    - Создать контроллер `ProductsController` с методом `Get`:
        ```csharp
        [HttpGet]
        public IEnumerable<ProductDto> Get()
        {
            return new List<ProductDto>
            {
                new ProductDto(1, "Book", 19.99m),
                new ProductDto(2, "Pen", 1.99m)
            };
        }
        ```
        
    - Запустить приложение и убедиться, что JSON-ответ содержит свойства `id`, `name`, `price`.
        
2. **Задача 2. Использовать `with`-выражение для обновления данных**  
    _Описание:_ Есть `record Employee(int Id, string FirstName, string LastName, decimal Salary)`. Получить новый объект с повышенной зарплатой на 10%.  
    _Ожидаемое решение:_
    
    ```csharp
    var emp = new Employee(1, "Anna", "Ivanova", 100000m);
    var empUpdated = emp with { Salary = emp.Salary * 1.1m };
    Console.WriteLine(empUpdated.Salary); // 110000
    ```
    
3. **Задача 3. Определить Value Object в доменной модели**  
    _Описание:_ В проекте для онлайн-магазина создайте `record Money(decimal Amount, string Currency)`. Продемонстрировать, как два объекта `Money` с одинаковыми значениями считаются равными.  
    _Ожидаемое решение:_
    
    ```csharp
    public record Money(decimal Amount, string Currency);
    var m1 = new Money(100, "USD");
    var m2 = new Money(100, "USD");
    Console.WriteLine(m1 == m2); // True
    ```
    

## Полезные советы и распространённые ошибки

- **Совет:** Используйте `record` для DTO в Web API, чтобы получить неизменяемые и легко сравнимые объекты.
- **Совет:** Если нужно наследование или абстрактная базовая модель, объявляйте `record class` вместо просто `record`.
- **Совет:** Применяйте `init`-свойства, если определяете `record` в классическом виде, чтобы сохранить иммутабельность.
- **Ошибка:** Ожидать, что `record` автоматически делает вложенные объекты иммутабельными. На самом деле, если свойство — изменяемый класс, его поля могут меняться.
- **Ошибка:** Использовать `record` там, где требуется изменяемая модель (например, Entity Framework Core сущности). EF Core пока не полностью поддерживает `init`-свойства для отслеживания изменений.

## Итоговый вывод (Резюме)

`record` в C# упрощает создание моделей данных, гарантирует сравнение по значению и поддерживает иммутабельность. Он особенно полезен при работе с DTO в Web API и при реализации Value Object в доменной модели. Правильное использование `record` повышает читаемость кода и снижает вероятность ошибок.
